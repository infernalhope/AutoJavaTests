vyvolání metody

Metody třídy ObjectExplorer:

  • invoke(classInstance, methodName, ... params)
  • invokeStatic(methodName, ... params)
  • invokeMethodException(Object studentClassInstance, String methodName, Object... methodParams)

Všechny 3 metody využívají vararg neboli speciálního typu zápisu parametru, který umožňuje metodě poslat proměnlivé množství parametrů. Tato možnost byla použita pro možnost předání proměnlivého počtu vstupů, které jsou nutné pro vyvolání studentovy metody. Každá metoda má jiný počet parametrů.

invoke

  • Metoda přijímá minimálně 3 parametry. První je instance studentovy třídy, druhým je String hodnota reprezentující požadovanou metodu (její název) a posledním je proměnlivý počet parametrů (viz. výše).
  • Metoda vrací datový typ Object, proto je nutné si v testovacích metodách tuto hodnotu přetypovat na požadovanou, respektive tu, kterou má studentova metoda vracet.
  • Může dojít k tomu, že nebude metoda vyvolána. Toto je blíže popsáno v sekci výjimky.

příklad
Studentova třída obsahuje metodu public int sumNumbers(int num1, int num2).
vyvolání metody bude vypadat tedy takto: (int)ObjectExplorer.invoke(classInstance, "sumNumbers", vstup1, vstup2);

________________________________________________________________________________

invokeStatic

  • Metoda má naprosto stejnou funkci jako metoda invoke, liší se pouze tím, že je uzpůsobena pro vyvolání statické metody.
  • Jinými slovy nepotřebuje instanci studentovy třídy, ze které se má požadovaná metoda vyvolat.
  • Může dojít k tomu, že nebude metoda vyvolána. Toto je blíže popsáno v sekci výjimky.

příklad
Studentova třída obsahuje metodu public static int sumNumbers(int num1, int num2).
vyvolání metody bude vypadat tedy takto: (int)ObjectExplorer.invokeStatic("sumNumbers", vstup1, vstup2);

________________________________________________________________________________

invokeMethodException

  • Tato metoda počítá s tím, že studentova metoda vyvolá výjimku a ty propaguje výše.
  • Návratová hodnota je Throwable, aby bylo možné zpracovat a vártit jakýkoli typ výjimky.
  • Může dojít k tomu, že nebude metoda vyvolána. Toto je blíže popsáno v sekci výjimky.

příklad
Studentova třída obsahuje metodu public static int sumNumbers(int num1, int num2) throws ArithmeticException.
vyvolání metody bude vypadat tedy takto: (ArithmeticException)ObjectExplorer.invokeMethodException("sumNumbers", vstup1, vstup2);
